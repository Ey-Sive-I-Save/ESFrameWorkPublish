# PlayMode 狼人杀测试复盘

本文件记录第一次狼人杀测试暴露的问题。结论是：AITalk 基础流程可跑通，但狼人杀这类强私密玩法必须严格区分公开信息、私密行动、主持者内部状态和玩家持续轮询义务。

## 流程验证成功

```text
1. 会话创建成功。
2. 等待玩家到齐成功。
3. 私密目录机制可用。
4. 阶段推进成功。
5. ACTION 格式可统计。
6. 最终回报成功。
```

## 暴露的问题

```text
1. 主持者公开消息泄露夜晚行动提交者。
2. PlayMode/Actions/ 根目录和文件名泄露私密行动者。
3. State 公开/私有边界不清。
4. 玩家 AI 写完一次声明、行动或发言后容易提前结束。
5. 主持者等待提示需要区分公开阶段和私密阶段。
```

## 已固化到规则的修复

```text
1. PlayMode 使用 State_Public.md 作为公开状态。
2. 主持者内部状态写入 PlayMode/Private/ModeratorAI/State_Internal.md。
3. 私密行动写入 PlayMode/Private/玩家AI名称/Actions/ 或 PlayMode/Actions/Private/玩家AI名称/。
4. 公开结算只写公开结果，不写私密行动提交者、行动原文、私密目标或私密结果。
5. 玩家未看到结束阶段前，必须持续轮询公开 State、主持者消息和自己的 Private 目录。
6. 等待私密行动时只能公开提示“仍有私密行动未提交”，不能点名身份或具体玩家。
7. 新增 Templates/PlayMode/狼人杀/ 专用模板。
```

## 后续测试重点

重新测试狼人杀时，重点检查：

```text
1. 公开 Messages 是否没有泄露 KILL/CHECK 提交者。
2. Public 和 State_Public 是否没有泄露身份、私密目标和查验结果。
3. Actions 根目录是否没有私密行动文件。
4. 预言家查验结果是否只进入预言家 Private。
5. 狼人击杀是否只公开死亡结果。
6. 玩家 AI 是否持续轮询直到主持者宣布结束。
```
