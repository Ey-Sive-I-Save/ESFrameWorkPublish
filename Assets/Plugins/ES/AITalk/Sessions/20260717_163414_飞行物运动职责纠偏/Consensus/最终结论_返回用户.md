# 最终结论_返回用户

本轮 AITalk 已形成第一阶段候选结论，但还需要用户最终拍板后才能进入代码实现。

## 核心结论

```text
飞行物运动模块不应成为“飞行物全包系统”。
它只负责运动模型、运动状态、Tick 推进、到达/失效、候选命中。
伤害、Buff、VFX、音效、对象池策略、技能释放和全局调度都不进入运动模块核心职责。
```

## 世界对象归属

```text
飞行物默认属于 Item，不属于 Entity。
运动只是 Entity 或 Item 的能力，不应被提升成独立世界对象身份。
不恢复 ESMotionCore，不新增 ItemMotionDomain。
```

第一阶段建议：

```text
Item
└── ItemBasicDomain
    ├── ItemMotionModule
    └── ItemProjectileModule
```

## 模块职责

```text
ItemMotionModule：
  第一阶段唯一位姿写回者，负责写 Transform / Rigidbody。

ItemProjectileModule：
  负责飞行物语义，调用 MotionSolver，维护飞行物运行状态，生成 ProjectileMotionResult 和 ProjectileHitCandidate。

ProjectileMotionSolver：
  纯 C# 运动求解，不继承 MonoBehaviour，不访问 Unity 组件，不写 Transform / Rigidbody。
```

## 结果语义

```text
MotionResultKind 只表达主运动状态：
None / Moving / Arrived / Expired / Blocked

HitCandidate 是独立伴随信息：
hasHitCandidate + ProjectileHitCandidate
```

这样允许：

```text
Moving + hasHitCandidate
Arrived + hasHitCandidate
Blocked + hasHitCandidate
```

不会把“主运动状态”和“候选命中”混成互斥枚举。

## 性能约束

```text
Tick 中不使用 LINQ、反射、字符串查找、GetComponent、Transform.Find。
Tick 中不 new 数组、List、class，也不扩容 NonAlloc 缓冲。
ItemProjectileModule 自持 NonAlloc 查询缓冲，容量不足时截断并记录调试信息。
第一阶段不做全局共享静态缓冲，避免重入和归属混乱。
```

## 第一阶段目录倾向

第一阶段优先保守放在 Item 内部，不急着创建 `Runtime/Movement` 根目录。

候选路径：

```text
Assets/Scripts/ESLogic/Runtime/Item/Item.cs
Assets/Scripts/ESLogic/Runtime/Item/Domains/Basic/_ItemBasicDomain.cs
Assets/Scripts/ESLogic/Runtime/Item/Domains/Basic/ItemBasicModules.cs
Assets/Scripts/ESLogic/Runtime/Item/Domains/Basic/ProjectileMotionTypes.cs
Assets/Scripts/ESLogic/Runtime/Item/Domains/Basic/ProjectileMotionSolver.cs
```

如果未来确实出现跨 `Entity / Item / Skill / Operation` 的真实复用，再考虑把纯 C# 求解层上提到：

```text
Assets/Scripts/ESLogic/Runtime/Shared/Motion/
```

不要一开始就创建过大的 `Runtime/Movement` 根目录。

## 第一阶段验证链路

```text
1. 创建一个 Item。
2. 挂 ItemBasicDomain、ItemMotionModule、ItemProjectileModule。
3. 启动直线飞行。
4. ProjectileModule 调用 MotionSolver.Step。
5. ProjectileModule 做 NonAlloc 候选命中检测。
6. ItemMotionModule 统一写回位姿。
7. 外部测试脚本只读取 ProjectileMotionResult。
```

验证目标：

```text
直线飞行能移动。
超时能 Expired。
到达目标能 Arrived。
碰到 Collider 能产生 HitCandidate。
不做伤害、Buff、VFX、音效、对象池回收。
```

```text
5. 是否会误碰 Entity KCC 热路径；第一阶段禁止。
```
