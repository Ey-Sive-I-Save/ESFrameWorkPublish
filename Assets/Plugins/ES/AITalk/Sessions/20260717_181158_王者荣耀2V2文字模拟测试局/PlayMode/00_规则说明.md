# PlayMode 规则说明：王者荣耀 2V2 地图文字模拟

## 定位

这是快节奏 MOBA 文字模拟，不完全复刻真实王者荣耀数值，但保留基本逻辑：

- 红蓝双方基地
- 每方外塔、高地塔、基地
- 中路兵线
- 野区资源点
- 击杀、推塔、团战
- 简化经济和等级
- 独立视野和私密战术

## 玩家规模

- 共 4 名玩家。
- 红方 2 名，蓝方 2 名。
- 玩家加入后自行选择队伍、英雄、定位、打法目标。
- 如果某队超过 2 人，主持者会要求后加入者改选队伍。
- 双方未形成 2V2 前不能正式开局。

## 地图

公开地图区域：

- 红方基地
- 红方高地塔
- 红方外塔
- 中路
- 蓝方外塔
- 蓝方高地塔
- 蓝方基地
- 上野区资源点
- 下野区资源点
- 河道草丛

## 简化数值

每名玩家默认：

- 等级：1
- 经济：300
- 状态：健康

每队默认：

- 外塔 HP：3
- 高地塔 HP：3
- 基地 HP：4

击杀奖励：

- 击杀者所在队伍经济 +200
- 被击杀者下一回合行动效果降低，除非主持者判定已复活

资源点奖励：

- 成功控制资源点的队伍获得经济 +150，且下一次团战判定 +1。

推塔：

- 集火攻击防御塔时，根据人数、兵线、资源优势、敌方干扰进行结算。
- 防御塔 HP 降为 0 时被摧毁。

## 阶段流程

1. 等待玩家到齐
2. 玩家选择队伍、英雄、定位、打法目标
3. 主持者确认双方阵容
4. 第一回合：开局分路与资源争夺
5. 第二回合：小规模冲突与推塔
6. 第三回合：团战与关键资源
7. 第四回合：高地/基地决战
8. 主持者结算胜负

主持者会按时间推进。玩家每回合提交行动后，主持者公开结算局势变化。鼓励玩家主动交战、绕后、控资源、推塔。

## 公开信息与独立视野

公开信息写入：

- PlayMode/State_Public.md
- PlayMode/Public/公共记录.md
- Messages/ 中 ModeratorAI 的公开消息

公开信息只包含所有人都能观察到的事实，例如：

- 当前阶段
- 存活/复活状态
- 公开地图局势
- 已被看见的交战结果
- 防御塔和基地血量
- 已公开爆发的团战

不公开的信息：

- 敌方私密行动原文
- 敌方视野外移动
- 埋伏、绕后、蹲草计划
- 主持者内部判定权重

玩家可把私密战术写入：

- PlayMode/Private/玩家AI名称/
- PlayMode/Actions/Private/玩家AI名称/

玩家禁止读取其他玩家 Private 目录和私密行动目录。

## 行动格式

每回合玩家必须提交机器可读行动。公开行动写入 PlayMode/Actions/Public/，私密行动写入自己的 PlayMode/Actions/Private/玩家AI名称/。

通用格式：

```text
ACTION: 动作名
TARGET: 目标
VALUE: 参数
REASON: 简短理由
```

允许动作：

```text
ACTION: MOVE
TARGET: 中路 / 上野区资源点 / 下野区资源点 / 河道草丛 / 敌方外塔 / 我方外塔
VALUE: 推线 / 支援 / 蹲草 / 绕后 / 防守 / 控资源
REASON: 简短理由
```

```text
ACTION: ATTACK
TARGET: 红方外塔 / 蓝方外塔 / 红方高地塔 / 蓝方高地塔 / 红方基地 / 蓝方基地 / 玩家名
VALUE: 集火推塔 / 消耗 / 强开 / 收割
REASON: 简短理由
```

```text
ACTION: TEAMFIGHT
TARGET: 中路 / 上野区资源点 / 下野区资源点 / 河道草丛 / 高地
VALUE: 开团 / 反打 / 拉扯 / 保护后排 / 切后排
REASON: 简短理由
```

```text
ACTION: FARM
TARGET: 中路兵线 / 上野区资源点 / 下野区资源点
VALUE: 发育 / 控线 / 抢资源
REASON: 简短理由
```

```text
ACTION: DEFEND
TARGET: 我方外塔 / 我方高地塔 / 我方基地
VALUE: 清线 / 守塔 / 反蹲
REASON: 简短理由
```

```text
ACTION: SKILL
TARGET: 玩家名 / 防御塔 / 资源点 / 区域
VALUE: 技能名或技能效果
REASON: 简短理由
```

```text
ACTION: COMBO
TARGET: 玩家名 / 多名玩家 / 防御塔
VALUE: 连招描述
REASON: 简短理由
```

玩家应尽量写出技能释放、控制、保护、惩戒、爆发、拉扯、追击或撤退细节。主持者会按英雄定位和技能意图结算团战效果。

## 结算方式

主持者按以下因素结算：

- 人数优势
- 英雄定位是否适配行动
- 兵线状态
- 防御塔状态
- 资源点优势
- 是否形成夹击或绕后
- 是否有队友配合
- 是否撞进敌方埋伏
- 行动理由是否清晰

主持者公开结算只写可见结果，不泄露未暴露的私密计划。

## 胜负条件

优先胜负条件：

- 摧毁敌方基地。

如果第四回合结束仍未摧毁基地，主持者按以下公开优势判定胜利：

- 防御塔数量
- 基地血量
- 击杀数
- 团战胜负
- 资源点控制
- 总体地图压制

## 玩家持续轮询义务

玩家不是写一条消息就结束。主持者宣布“游戏结束”前，玩家必须持续轮询：

- PlayMode/State_Public.md
- Messages/ 中 ModeratorAI 新消息
- 自己的 PlayMode/Private/玩家AI名称/
- 自己的 PlayMode/Actions/Private/玩家AI名称/

提交行动后不要结束任务；继续等待主持者结算和下一阶段指令。
