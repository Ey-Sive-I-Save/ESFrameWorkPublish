# ES 开发管理超级窗口

## 📋 功能概述

ES 开发管理超级窗口是一个类似 Notion 的强大开发管理工具，集成在 Unity 编辑器中，用于管理：
- 📝 **开发日志** - 记录所有开发活动
- 🎯 **开发计划** - 管理项目计划和任务
- 📅 **排期管理** - 安排和跟踪日程
- 🎖️ **目标管理** - 设定和追踪开发目标
- 📊 **统计分析** - 数据可视化和分析

## 🚀 快速开始

### 打开窗口
在 Unity 编辑器菜单中选择：
```
Tools > ES工具 > ES开发管理
```

### 首次使用
1. 窗口会自动创建数据资产文件在：`Assets/ES/DevManagement/DevManagementData.asset`
2. 系统会生成示例数据帮助您快速上手
3. 所有数据自动保存，无需手动操作

## 📖 功能详解

### 📊 仪表板
- 查看所有数据的统计概览
- 快速访问最近活动
- 一键创建新条目的快捷操作

### 📝 开发日志
**功能：**
- 记录日常开发活动
- 支持多种日志类型：功能开发、Bug修复、性能优化、代码重构等
- 设置优先级和标签
- 自动记录创建和修改时间

**使用方法：**
1. 点击 `📝 开发日志 > 新建日志`
2. 填写标题和内容
3. 选择类型和优先级
4. 可添加多个标签用于分类
5. 点击 `✅ 创建日志` 按钮

**查看日志：**
- 在 `日志列表` 中查看所有日志
- 支持表格视图，可直接编辑
- 自动排序和筛选

### 🎯 开发计划
**功能：**
- 管理长期和短期开发计划
- 跟踪计划进度（0-100%）
- 设置开始和结束日期
- 支持多种状态：未开始、进行中、已完成、已暂停、已取消

**使用方法：**
1. 点击 `🎯 开发计划 > 新建计划`
2. 输入计划名称和描述
3. 设置时间范围
4. 选择状态和优先级
5. 调整进度滑块
6. 点击 `✅ 创建计划` 按钮

**进度管理：**
- 在计划列表中直接拖动进度条
- 实时更新计划状态
- 自动计算完成率

### 📅 排期管理
**功能：**
- 创建具体的日程安排
- 设置任务时间段（开始时间 - 结束时间）
- 分配负责人
- 标记完成状态

**使用方法：**
1. 点击 `📅 排期管理 > 新建排期`
2. 填写任务名称和描述
3. 选择排期日期
4. 设置开始和结束时间
5. 填写负责人（可选）
6. 点击 `✅ 创建排期` 按钮

**日历视图：**
- 在排期日历中查看所有安排
- 快速筛选今日、本周排期
- 勾选完成状态

### 🎖️ 目标管理
**功能：**
- 设定长期开发目标
- 跟踪目标完成进度
- 多种目标类型：功能目标、性能目标、质量目标、学习目标、里程碑
- 支持目标延期和取消

**使用方法：**
1. 点击 `🎖️ 目标管理 > 新建目标`
2. 输入目标名称和详细描述
3. 设置截止日期
4. 选择目标类型和状态
5. 设置优先级
6. 调整完成进度
7. 点击 `✅ 创建目标` 按钮

**目标追踪：**
- 实时更新进度条
- 自动提醒延期目标
- 查看目标完成统计

### 📈 统计分析
**功能：**
- 自动统计所有数据
- 显示计划完成率
- 本周排期统计
- 目标达成率分析

**数据包括：**
- 总日志/计划/排期/目标数量
- 已完成/进行中/未开始计划分布
- 本周排期完成情况
- 活跃目标数量
- 完成率百分比（带进度条显示）

### ⚙️ 设置
**功能：**
- 查看数据文件路径
- 手动保存数据
- 重新加载数据
- 定位数据资产

**危险操作：**
- 清空所有日志
- 清空所有计划
- 重置所有数据（需二次确认）

## 💾 数据管理

### 自动保存
- 所有操作自动保存到 ScriptableObject
- 无需担心数据丢失
- 支持版本控制（可提交到 Git）

### 数据位置
默认数据文件：`Assets/ES/DevManagement/DevManagementData.asset`

### 手动备份
1. 在 Project 窗口中找到数据文件
2. 右键选择 `Duplicate` 创建备份
3. 或复制到其他位置保存

### 数据导出
未来版本将支持：
- 导出为 Markdown
- 导出为 Excel
- 导出为 JSON

### 过滤和搜索
- 使用菜单顶部搜索栏快速定位

## ⚠️ 注意事项

1. **数据安全**
   - 定期备份数据文件
   - 重置数据前请三思
   - 建议加入版本控制

2. **性能考虑**
   - 单个数据资产建议不超过 1000 条记录
   - 大量数据时可创建多个数据文件
   - 定期清理过期数据

3. **团队协作**
   - 数据文件可通过 Git 共享
   - 注意合并冲突
   - 建议每人使用独立数据文件

## 🐛 问题排查

### 窗口打不开
- 检查是否有编译错误
- 重启 Unity 编辑器
- 删除 Library 文件夹重新编译

### 数据丢失
- 检查数据文件是否存在
- 在设置页面重新加载数据
- 恢复备份文件

### 表格显示异常
- 刷新窗口（F5 或刷新按钮）
- 重新打开窗口
- 检查 Odin Inspector 版本
