# 🚀 ES开发管理工具 - 快速入门

## 📌 5分钟快速上手

### 第一步：打开工具
在Unity顶部菜单选择：**Tools → ES工具 → ES开发管理 V2**

### 2️⃣ 日志管理 - 记录开发过程

#### 🆕 新建日志
1. 点击左侧 **"开发日志/新建"**
2. 填写标题和内容
3. 选择类型（功能开发/Bug修复/优化等）
4. 设置优先级
5. 添加标签
6. 点击 **"创建"** 按钮

#### 📋 查看日志
- **列表视图**: 筛选和搜索
- **详情页**: 查看完整信息
- **编辑**: 点击详情页的 **"⚙️ 编辑"** 按钮

#### 💡 小技巧
- 使用 **搜索框** 快速查找
- 用 **标签** 分类管理
- **优先级** 帮助排序

---

### 3️⃣ 任务管理 - 高效协作

#### 🆕 新建任务
1. 点击 **"任务看板/新建"**
2. 填写任务名称和描述
3. 设置开始和截止日期
4. 分配负责人
5. 添加子任务清单
6. 点击 **"创建任务"**

#### ✅ 子任务清单
```
☐ 子任务1 - 张三
☐ 子任务2 - 李四
☐ 子任务3 - 王五

进度: ███░░░░░ 30%
```

**操作**:
- ✓ 勾选框标记完成
- ➕ 添加新子任务
- 🗑️ 删除子任务
- ↕️ 拖拽排序

#### 📊 看板视图
```
开始 → 进行中 → 评估中 → 已完成
```
直观查看任务流转

---

## 🆘 常见问题

### Q: 数据保存在哪里？
**A**: `Assets/ES/DevManagement/DevManagementDataV2.asset`

### Q: 如何多人协作？
**A**: 数据文件可以加入Git等版本控制，团队共享

### Q: 数据丢失了怎么办？
**A**: 
1. 检查数据文件是否存在
2. 点击"设置"→"定位文件"
3. 如果文件损坏，从备份恢复

### Q: 如何备份数据？
**A**: 
1. 复制数据文件
2. 或者导出Markdown报告
3. 定期提交到版本控制

### Q: 任务进度如何计算？
**A**: 自动根据子任务完成情况计算
```
进度 = (已完成子任务数 / 总子任务数) × 100%
```

### Q: 能否自定义类型和状态？
**A**: 当前版本为预设，后续版本可能支持自定义
